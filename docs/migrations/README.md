# Migration Notes

Add one file per response-contract change.

Filename format:

- `YYYY-MM-DD-vX.Y.Z-short-title.md`

Each migration note should include:

1. Affected tool names.
2. Summary of breaking/non-breaking behavior.
3. `before` and `after` payload examples.
4. Agent update guidance.

Use `template.md` as the starting point.
